# Инструкция по настройке Coolify для исправления ошибки

## Проблема: `/bin/bash: -c: option requires an argument`

Эта ошибка обычно возникает из-за неправильных настроек healthcheck в Coolify.

## Решение:

### 1. Отключите Healthcheck в настройках Coolify

1. Откройте ваше приложение в Coolify
2. Перейдите в раздел **"Configuration"** (Конфигурация)
3. Найдите раздел **"Healthcheck"** или **"Health Check"**
4. **Отключите healthcheck** или оставьте поля пустыми
5. Для Telegram бота с polling healthcheck не требуется, так как бот не слушает HTTP порт

### 2. Проверьте переменные окружения

1. В разделе **"Configuration"** найдите **"Environment Variables"**
2. Убедитесь, что установлена переменная:
   - **Имя**: `BOT_TOKEN`
   - **Значение**: ваш токен Telegram бота (например: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 3. Проверьте настройки портов

1. В разделе **"Configuration"** найдите **"Ports"**
2. Для Telegram бота с polling порты **не требуются**
3. Убедитесь, что нет обязательных портов в настройках

### 4. Пересоберите и перезапустите

1. Нажмите кнопку **"Redeploy"** (Пересобрать)
2. Дождитесь завершения сборки
3. Проверьте логи - должно появиться сообщение "Бот запускается..."

### 5. Если проблема сохраняется

Проверьте логи в разделе **"Logs"**:
- Если видите "ОШИБКА: BOT_TOKEN не установлен!" - проверьте переменные окружения
- Если видите другие ошибки - скопируйте их и проверьте настройки

## Альтернативное решение (если healthcheck обязателен)

Если в вашей версии Coolify нельзя отключить healthcheck, можно использовать простую команду:

**В настройках Healthcheck в Coolify:**
- **Command**: `python -c "import sys; sys.exit(0)"`
- **Interval**: `30s`
- **Timeout**: `10s`
- **Start Period**: `40s`
- **Retries**: `3`

Но для Telegram бота лучше полностью отключить healthcheck.
